# X Publishing Checklist

Xに投稿する前に確認するチェックリストです。

## 情報安全性

- [ ] APIキー、トークン、パスワードが含まれていない
- [ ] 秘密鍵、認証情報、内部URLが含まれていない
- [ ] 個人情報、顧客情報、契約情報が含まれていない
- [ ] 未公開ロードマップや内部事情が含まれていない
- [ ] スクリーンショットに秘密情報が映っていない

## 内容品質

- [ ] 最初の1文で何の話か分かる
- [ ] 読者にとっての価値が明確
- [ ] 主張が大きすぎない
- [ ] 事実と意見が混ざりすぎていない
- [ ] 誤解を招く断定をしていない
- [ ] 必要なら元記事やGitHub URLを添えている

## X向け表現

- [ ] 1投稿で言いたいことが1つに絞られている
- [ ] 長すぎる場合はスレッド化している
- [ ] 冒頭に具体性がある
- [ ] 最後に読者の次の行動がある
- [ ] ハッシュタグを付けすぎていない

## 投稿NGの例

```text
本番環境の運用手順そのもの
社内でしか知らない障害情報
顧客名を含む成果報告
未公開機能の詳細
認証情報が映ったスクリーンショット
```
